# チェスウェブアプリ 仕様書

## 1. プロジェクト概要

ブラウザ上で動作するチェスウェブアプリ。ユーザーはCPU AI と対戦できる。  
シンプルな操作性と Apple ライクな落ち着いたデザインを目指す。

---

## 2. 画面構成

### 2.1 画面一覧

| 画面名 | 説明 |
|--------|------|
| ホーム（スタート） | アプリの起点。ゲーム開始・ルール確認へ遷移 |
| 設定 | 駒の色（白/黒）と AI 難易度を選択 |
| プレイ | チェス盤・手番表示・手の履歴を含むゲーム画面 |
| ルール確認 | チェスの基本ルールを説明するページ |

### 2.2 画面遷移フロー

```
ホーム
├── [ゲームを始める] → 設定 → プレイ → ホーム（ゲーム終了後）
└── [ルールを確認する] → ルール確認 → ホーム
```

---

## 3. 機能仕様

### 3.1 ホーム画面

- 「ゲームを始める」ボタン：設定画面へ遷移
- 「ルールを確認する」ボタン：ルール確認画面へ遷移

### 3.2 設定画面

| 項目 | 内容 |
|------|------|
| 駒の色選択 | 白（先手）または 黒（後手）を選択する |
| AI 難易度選択 | 初級 / 中級 / 上級 の 3 段階 |
| 開始ボタン | 設定を確定してプレイ画面へ遷移 |

### 3.3 プレイ画面

#### 3.3.1 チェス盤

- 8×8 マスのチェス盤を表示する
- 選択した駒が移動可能なマスをハイライト表示する
- 直前の指手（移動元・移動先）をハイライト表示する

#### 3.3.2 ゲームロジック

**実装するルール**

- 全駒の基本移動（キング、クイーン、ルーク、ビショップ、ナイト、ポーン）
- チェック検出（王手状態を表示）
- チェックメイト（詰み）検出によるゲーム終了
- ステイルメイト（引き分け）検出によるゲーム終了

**省略するルール**

- キャスリング（ショート・ロング）
- アンパッサン
- ポーンのプロモーション（昇格）

#### 3.3.3 手の履歴パネル

- 代数記法（例：`e2-e4`）で指手リストを表示する
- 手番ごとに白・黒の指手を並べて表示する

#### 3.3.4 ゲーム終了

- 勝敗または引き分けが確定したとき、結果をモーダルまたはバナーで表示する
  - 表示内容：「あなたの勝ち」「あなたの負け」「引き分け」
- ホーム画面に戻るボタンを提供する

### 3.4 ルール確認画面

- チェスの基本ルールをテキストと図で説明する
- 各駒の動き方を一覧で示す
- ホーム画面へ戻るボタンを配置する

## 4. CPU AI 仕様

| 難易度 | 説明 | Stockfish 設定の目安 |
|--------|------|----------------------|
| 初級 | 弱い手を指すこともある。初心者向け | Skill Level 1–3 |
| 中級 | 標準的な強さ。ある程度の読みあり | Skill Level 8–12 |
| 上級 | 強い手を安定して指す。上級者向け | Skill Level 18–20 |

- AI エンジンとして **Stockfish（WebAssembly 版）** を使用する
- AI の思考時間は各手 1〜3 秒程度とする（難易度により調整）
- AI の手番中はローディングインジケーターを表示する

## 5. UI/UX 方針

### 5.1 デザインコンセプト

- **Apple ライク**：余白を大きく取り、角丸・シャドウを活用したクリーンなデザイン
- **落ち着いたトーン**：ニュートラルカラーをベースに、アクセントカラーを控えめに使用
- **親しみやすさ**：フォントは読みやすいサンセリフ体を採用

### 5.2 操作性

- 駒の移動はクリック操作のみ（ドラッグ&ドロップは任意）
- 最低限のステップでゲームを開始できる（ホーム → 設定 → プレイ）
- モバイルブラウザにも対応するレスポンシブレイアウトを採用する

### 5.3 カラーパレット（参考）

| 用途 | 色 |
|------|----|
| 背景 | `#F5F5F7`（ライトグレー） |
| 盤面（白マス） | `#FFFEFE` |
| 盤面（黒マス） | `#6B7280` |
| アクセント | `#0071E3`（Apple Blue） |
| テキスト（メイン） | `#1D1D1F` |

## 6. 技術スタック

| 分類 | 採用技術 |
|------|----------|
| フロントエンドフレームワーク | React + TypeScript |
| チェスエンジン | Stockfish（WebAssembly）|
| チェスロジックライブラリ | chess.js（ルール判定・FEN 管理）|
| スタイリング | CSS Modules または Tailwind CSS |
| ビルドツール | Vite |

---

## 7. 非機能要件

- **対応ブラウザ**：最新版の Chrome / Firefox / Safari / Edge
- **レスポンシブ対応**：スマートフォン（375px〜）からデスクトップ（1440px）まで
- **パフォーマンス**：初回読み込み 3 秒以内（Stockfish の WASM ファイルは遅延ロード）
- **データ永続化**：なし（ページリロードでゲームはリセット）

---

## 8. 対象外（スコープ外）

- オンライン対人対戦
- ユーザー登録・ログイン
- 対局の保存・読み込み
- キャスリング・アンパッサン・プロモーションなどの特殊ルール
- タイマー機能
